## Key Patterns to Follow

### Angular Frontend Rules

1. **All components:** `standalone: true`, no NgModules
2. **Control flow:** Angular 20 syntax — `@if`, `@else if`, `@else`, `@for`, `@let`
3. **Typed forms:** `new FormGroup({ field: new FormControl<T>(default, { nonNullable: true }) })`
4. **No `any` types** in production code (strict TypeScript)
5. **CSS design tokens only** — no hardcoded colours/spacing; see `docs/DESIGN.md §3`
6. **HTTP interceptor** attaches JWT; clears token + redirects on 401/403
7. **`provideHttpClient()` + `provideHttpClientTesting()`** in specs — NOT `HttpClientTestingModule`
8. **Provider ordering in specs:** always put `provideRouter([])` BEFORE `{ provide: ActivatedRoute, useValue: routeMock }` — otherwise `provideRouter` overrides the mock (learned the hard way in WU-07)
9. **Subscriptions:** `takeUntil(this.destroy$)` pattern; complete in `ngOnDestroy()`
10. **Drawers** are 400px right-slide panels, shown inline with `@if (showDrawer)`, NOT separate routes

### Test Patterns

```typescript
// Mutable route mock (avoids TestBed.overrideProvider error)
const routeMock = { snapshot: { paramMap: convertToParamMap({}) } };
// Mutate in inner beforeEach BEFORE buildFixture(), AFTER outer beforeEach configures TestBed

// Spy objects
let serviceSpyObj = jasmine.createSpyObj('ServiceName', ['method1', 'method2']);
serviceSpyObj.method1.and.returnValue(of(data));

// Contact autocomplete blur test
fakeAsync + tick(150)  // for setTimeout(150) in onContactBlur / onCompanyBlur

// Search debounce test
fakeAsync + tick(300)  // for debounceTime(300) in search valueChanges
```

### Backend Rules

1. **Architecture:** Controller → Service → Repository only
2. **Redis cache-first (hard constraint NFR-P03):**
   ```java
   // Every read:
   Object cached = redis.get(key);
   if (cached != null) return cached;
   Object fromDb = repository.findById(id);
   redis.set(key, fromDb, TTL_24H);
   return fromDb;
   // Every write: immediately invalidate all affected keys
   ```
3. **Both cache-hit AND cache-miss paths must be tested** (from `bd memories`)
4. **Error responses:** always structured JSON `{ "error": { "code", "message", "fields?" } }` — never HTML
5. **JWT:** 8h expiry, secret from env var, never logged
6. **Bean Validation (`@Valid`)** on all request bodies

### Domain Model Quick Reference

```
Contact: id, name(first+last), email, phone, jobTitle, company(id,name), owner(id,name,email), tags[], createdAt
Company: id, name (read-only lookup)
Deal: id, title, value(decimal), stage(enum), expectedClose(date), contact(id,name), owner(id,name,email), createdAt
Activity: id, type(CALL|EMAIL|MEETING|NOTE), subject, notes, occurredAt, contact(opt), deal(opt), author(id,name)
Task: id, title, description, dueDate, completed, assignee(id,name), contact(opt), deal(opt), createdAt
Tag: id, name, colour(hex #rrggbb)
```
