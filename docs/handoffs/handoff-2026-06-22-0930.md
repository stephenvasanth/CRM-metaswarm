## What Was Done This Session

### 1. Backend caching architecture fixed
Removed `findAll` caching everywhere. Redis now only caches:
- `findById` for contacts and deals (TTL 24 h, key `contacts:id:<id>` / `deals:id:<id>`)

## Key Architecture Decisions

- **Redis:** Cache ONLY `findById` (contacts, deals) and `deals:stats`. Never cache list/paginated endpoints. Specific key eviction only — no KEYS pattern scanning.
- **JWT:** 8-hour expiry, stored in localStorage, attached via HTTP interceptor, cleared + redirect on 401/403.
- **Angular:** All components `standalone: true`. Lazy-loaded feature modules. `strict: true` TypeScript.
- **Hard delete for Contacts:** Deletes the contact + directly-linked Activities. Tasks are preserved with contact reference cleared.
- **Company management deferred:** Companies are read-only id+name lookup for Contact picker only. Full company CRUD is out of scope.
- **Activity editing deferred:** Users can delete but not edit activities.
- **Drawers:** 400px right-slide for Deal form, Activity form, Task form, Tag creation. Full pages for Contact create/edit and Contact detail.
- **Optimistic UI:** Task complete toggle and Kanban drag-and-drop both revert on server error with a toast.
- **Design tokens:** All CSS uses custom properties from `docs/DESIGN.md §3`. No hardcoded values.
